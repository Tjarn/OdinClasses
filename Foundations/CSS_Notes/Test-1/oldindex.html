CSS -- What it is

In order for me to fully understand something, I usually need to build a mental model of some sort. That's what I'm doing here. I hope that by writing things out, I can nail down the concepts hard enough that they stick long enough to use. I'll be messing around with styling in here, so hope to have some demos that I can mess with.

What is it? From what I can see from this point in my learning is that it's a method of selecting html entities and adding specifications as to how they render. It looks like it is in the same place as PostScript is with printing in speaking to a web browser. The computer pumps out encoded binary streams to either, one renders graphical text on a printer, the other renders graphics and text on a browser screen. Of course the printed rendition is static and the other is dynamic, so the parallel is limited. But for now it looks like a way to add 'HOW' to html's WHAT.

More specifically it is a number of methods of identifying, marking, and combining discrete html entities and specifying ways to render them. From there, it looks to get complicated fast.

Here's a selection from a random CSS file in my Linux system:

table.navigation { background: #ffeeee; border: solid 1px #ffaaaa; margin-top: 0.5em; margin-bottom: 0.5em; } .navigation a { color: #770000; } .navigation a:visited { color: #550000; } .navigation .title { font-size: 200%; }

Basically, it consists of a line that runs through a complicated pattern matcher, and when matched delivers the contents of the {} delimited block that follows the pattern line. The contents of the block are one or more semicolon (;) terminated lines. These lines are colon (:) split. Most seem to set some sort of displayed property.

I need to take some of these patterns apart to see how they work.

Let's start with the simplest. '*' asterisk, star, splat. It matches anything like in sh or bash. if we had the css lines:

...} p { font-size: 26Px} ...

A common container is <p> - and that won't show up in a text editor. But that's what the p matches. It's like it hollers to all the 'p's out there SET YOUR FONTS TO 26! In this document I'm using the container <p class="body-text"> for all normal paragraphs. I should be able to match those containers with the line p.body-text. So the code to set the font-size to 26 pixels would be:

p.body-text { font-size: 26px; }

So I tried it and it worked exactly as advertised.
