Ordered and Unordered Lists

In this section I'll try a couple of lists. There are two types of lists that we need to consider, ordered and unordered. First, The commanalities:

One: Both types of lists are bounded by tags as are most other html objects. Both lists are two letter tags ending in 'l'.

Two: Each item in a list starts with the <li> tag and ends with the </li> tag.

Unordered Lists

Unordered lists are bounded by the tag pair <ul> and </ul>. Within the list, each element is bounded by the tag pair <li> and </li>. I would expect this to mean that you can change an ordered list to an unordered list by changing the enclosing list tags.

As an example of an unordered list, let's try to set one up inside this paragraph. Look at the source of this page to see the list after this text.

  • Hamburger
  • French Fries
  • Milkshake
  • Hot Peppers

The list worked just fine, and each element recieved a bullet prefix as a format. Now let's try the same list outside the bounds of the current paragraph.

  • Hamburger
  • French Fries
  • Milkshake
  • Hot Peppers

The previous list is before this paragraph and after the last. I don't see any difference whether it's enclosed in paragraph tags or not.

Ordered Lists

To make an ordered list, enclose the list elements in the >ol< and >/ol< tags.

  1. Hamburger
  2. French Fries
  3. Milkshake
  4. Hot Peppers

The obvious difference is the change of the bullet points to a sequential list of items. I'm going to mess around for a while and see what happens when you put lists inside list items. Being lazy, I'll just use the list items I've already typed in.
